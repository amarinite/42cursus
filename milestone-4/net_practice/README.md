# NetPractice

*This project has been created as part of the 42 curriculum by amarquez.*

## Description

NetPractice is a network configuration training tool designed to teach fundamental networking concepts through practical exercises. The project consists of 10 progressive levels where you configure network interfaces, routers, and routing tables to establish proper communication between devices. Each level increases in complexity, requiring understanding of IP addressing, subnet masks, routing, and network topology.

The goal is to master essential networking concepts by solving configuration challenges that simulate real-world network scenarios.

## Instructions

### Running the Training Interface

1. Open `index.html` in a web browser (due to technical constraints, a Chromium browser is strongly recommended)
2. Configure the network parameters as needed
3. Click "Check again" to validate your configuration
4. Once successful, click "Get my config" to export the configuration file
5. Click "next" to advance to the next level

### Exporting Configurations

- After successfully completing each level, export the configuration using the "Get my config" button
- Each exported JSON file will be named with its corresponding level number
- Don’t forget to enter your login in the training interface.

### Submission Requirements

- Place all 10 exported configuration files at the root of your repository
- Ensure each file corresponds to a successfully completed level
- The repository must contain: `level1.json`, `level2.json`, ..., `level10.json`

## Resources

### Networking Concepts Studied

This project covers essential networking concepts including:

- **TCP/IP Addressing**: Understanding IPv4 address structure and binary notation
- **Subnet Masks**: CIDR notation, network/host portions, and subnetting calculations
- **Default Gateway**: Routing traffic between different networks
- **Routers and Switches**: Device roles and packet forwarding
- **OSI Model Layers**: Focus on Network Layer (Layer 3) routing and addressing
- **Routing Tables**: Static route configuration and destination matching
